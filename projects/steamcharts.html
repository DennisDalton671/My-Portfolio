About SteamCharts

SteamCharts was inspired by web-based tracking sites, and I wanted to develop a mobile-first alternative to check real-time player statistics for Steam games. This was also one of the first apps I built using Android Studio and Kotlin. It was my first time working with databases to store and access game-related data, and it was also my first experience using APIs to pull live data for users.

One of the biggest challenges I faced was parsing Steamâ€™s API data. Their JSON structure is somewhat nested, meaning the actual game information (like player count, description, and pricing) was buried within multiple layers of data. Figuring out how to extract the relevant data efficiently took some time, but overcoming this challenge improved my understanding of handling APIs and JSON data in Android development.
